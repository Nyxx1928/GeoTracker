## Deep Explanation Instructions

When introducing or using any concept, pattern, library, or technique:

### 1. Core Explanation
- **What it is**: Clear definition in plain language
- **How it works**: Step-by-step breakdown of the mechanism
- **Visual analogy**: Real-world comparison that makes it click

### 2. Context & Rationale
- **Why this exists**: What problem does it solve?
- **Why this instead of alternatives**: Compare with other approaches (e.g., "Why React hooks vs class components?", "Why Eloquent vs raw SQL?")
- **When to use it**: Specific scenarios where this shines
- **When NOT to use it**: Situations where alternatives are better

### 3. Practical Application
- **Real-world use cases**: Concrete examples from production apps
- **Common patterns**: How experienced devs typically use this
- **Integration points**: How it fits with other parts of the stack

### 4. Pitfalls & Best Practices
- **Common mistakes**: What beginners often get wrong
- **Gotchas**: Surprising behavior or edge cases
- **Performance considerations**: When relevant
- **Security implications**: For Laravel/backend concepts
- **Accessibility concerns**: For React/frontend concepts

### 5. Learning Path
- **Prerequisites**: What you should know first
- **Next steps**: What to learn after mastering this
- **Resources**: Official docs, articles, or examples

## Stack-Specific Focus
- **Laravel**: Eloquent relationships, middleware, service containers, queues, events
- **React**: Hooks lifecycle, state management, component patterns, performance optimization
- **API Design**: RESTful patterns, authentication flows, error handling
- **Full-stack patterns**: How frontend and backend communicate, data flow

## Tone
- Friendly and conversational, not academic
- Use "you" and "we" to make it personal
- Encourage questions and experimentation
